# CapstoneProject Repository

## Overview
This repository contains all the relative files I have created for my senior capstone project at Eastern University. The project itself was a cloud based data pipeline involving certain cryptocurrency and stock information. I used CoinGecko's API to obtain the cryptocurrency information and Finnhub's API to obtain the stock information. Based on the call rate limits for the API's I had to choose 10 coins to examine in this project, as well as 7 stocks. I had chosed certain stocks to examine potential relationships between certain coins and stocks. After the database was built and being populated on a schedule, a Grafana dashboard was built to visualize the data allowing for the user to draw insights.

## System Architecture
The system retrieves cryptocurrency market data from the CoinGecko API and Finnhub API using Python ingestion scripts hosted on AWS Lambda to run on fixed schedules that were calculated to ensure compliance with API call limits. Data is processed and stored in a PostgreSQL database hosted on AWS RDS. Grafana connects to the database to provide interactive visualizations and market analysis for the user.

## Database Design
The database is built on PostgreSQL and hosted on AWS RDS, organized around a central assets table that links to type-specific tables for cryptocurrencies and stocks, as well as five time-series history tables tracking price, volume, market cap, change metrics, and OHLC data at 15-minute intervals. A separate global_crypto_stats table captures hourly macro-level market data. All time-series tables are indexed on (asset_id, recorded_at) to support efficient time-based querying for the Grafana dashboard.

## Data Collection/Ingestion
Data is collected through four automated Python scripts deployed as AWS Lambda functions and triggered on scheduled intervals using Amazon EventBridge. Cryptocurrency market data including price, volume, market capitalization, and 24-hour change metrics for 10 coins is pulled from the CoinGecko API every 15 minutes, while all-time high and low values and market rankings are updated conditionally only when a change is detected. Stock data including price, volume, OHLC, and 52-week high and low values for 7 equities is pulled from the Finnhub API every 15 minutes during market hours on weekdays. Global crypto market statistics are collected hourly from CoinGecko's global endpoint. A fifth Lambda function runs daily to enforce data retention policies, removing records older than 90 days from time-series tables and older than 30 days from the change metrics table, keeping storage usage stable over time.

## Dashboard and Visualizations
Data is visualized through a multi-tab Grafana Cloud dashboard connected directly to the AWS RDS PostgreSQL database as an external data source. The dashboard is organized into six tabs covering a market overview, crypto deep dive, stock deep dive, crypto versus stock comparison, market structure analysis, and an interactive asset explorer. The market overview tab provides a high-level snapshot of the crypto market including total market capitalization, trading volume, Bitcoin dominance, top gainers and losers, and normalized price and volume correlation charts across all tracked assets. The market structure analysis tab examines the tracked cryptocurrency universe through three analytical lenses — stablecoin versus non-stablecoin, large cap versus small cap, and native chain versus multi-chain tokens — comparing price stability, volatility, and market dominance across each grouping. The crypto versus stock comparison tab uses price normalization to a common baseline of 100 to enable direct performance comparison between assets regardless of their actual price levels. The interactive asset explorer tab allows users to select any tracked asset from a dropdown menu and view asset-specific visualizations including price history, volume, market cap, OHLC candlestick charts, and proximity to all-time highs or 52-week highs depending on asset type

## Known Limitations
Several limitations exist in the current implementation that are worth noting. Stock OHLC data is sourced from Finnhub's quote endpoint which returns the day's running high and low rather than true interval-specific values, meaning candlestick charts reflect daily price ranges rather than 15-minute candle intervals. Stock trading volume is represented as a 10-day average from Finnhub's metrics endpoint rather than real-time volume, as real-time volume is not available on the free tier. Due to the 15-minute polling interval there are occasional brief moments where a stock's current price exceeds the stored 52-week high during periods of rapid upward movement, as the rolling 52-week window updates continuously on Finnhub's end between script runs. The CoinGecko free Demo tier imposes a limit of 10,000 API calls per month, which constrains the frequency at which certain endpoints can be polled and was a key factor in the decision to collect crypto OHLC data on a 2-hour interval rather than every 15 minutes. Finally, the global crypto market capitalization and total stock market capitalization cannot be compared directly in this project as no free API provides a true total stock market capitalization figure equivalent to CoinGecko's global crypto endpoint, so the global stats table tracks crypto market data only.

## Future Improvements
Several enhancements could be made to extend the scope and depth of this project given additional time or resources. Upgrading to a paid CoinGecko API tier would enable true 15-minute interval OHLC data for cryptocurrencies, removing the current 2-hour limitation and enabling consistent candlestick chart granularity across both asset classes. Expanding the tracked asset universe to include additional cryptocurrencies and stocks across multiple sectors and exchanges would strengthen the comparative analysis and make the market structure findings more statistically meaningful. Incorporating a crypto fear and greed index as an additional data source would add a sentiment layer to the dashboard, allowing correlations between market sentiment and price movement to be explored. Adding real-time price alerts via AWS SNS triggered when an asset crosses a configurable threshold would increase the practical utility of the pipeline beyond passive monitoring. Replacing the 10-day average stock volume with a real-time volume feed would improve the accuracy of the volume comparison visualizations. Finally, implementing a machine learning model to identify patterns in the time-series data and generate short-term price movement predictions would be a natural extension of the data infrastructure already in place.
